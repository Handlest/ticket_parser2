# Бот отслеживания авиабилетов

Сервис периодически опрашивает сайты авиакомпаний по заданным параметрам и
присылает в Telegram уведомление, когда находит билет дешевле заданного порога.
Управление — через Telegram-бота. Хранилище — SQLite. Запуск — одной командой
в Docker.

## Возможности

- Добавление отслеживаний с параметрами: город отправления, город назначения,
  дата, порог цены, категория льготы, интервал проверки.
- Уведомления в Telegram при нахождении подходящего билета (цена + ссылка).
- Автоудаление отслеживаний с прошедшей датой вылета.
- Поддержка льготных категорий (см. [benefits.md](benefits.md)).
- Лёгкое подключение новых сайтов-провайдеров (см. [providers.md](providers.md)).

## Технологический стек

- **Python 3.12**
- **aiogram 3** — Telegram-бот (async, FSM).
- **Playwright** — рендеринг JS-сайтов (современная замена Selenium).
- **httpx + selectolax** — лёгкие провайдеры (API/статичный HTML).
- **APScheduler** — периодические проверки.
- **SQLite + SQLAlchemy 2.0 (async)** — хранилище. Миграций нет: таблицы
  создаются на старте.
- **PyYAML + pydantic** — конфигурация из `setup.yaml`.
- **Docker / docker-compose** — запуск.

## Архитектура

```
app/
├── main.py            # точка входа: бот + планировщик
├── config/            # загрузка и валидация setup.yaml
├── domain/            # категории льгот
├── db/                # модели, сессия, репозиторий (SQLite)
├── providers/         # парсеры сайтов (base + registry + s7/redwing)
├── core/              # планировщик проверок и матчинг по цене
├── notifications/     # отправка уведомлений в Telegram
└── bot/               # хендлеры, FSM, клавиатуры, middleware
```

Ядро работает только с нормализованными `SearchQuery` / `FlightOffer` и не знает
о конкретных сайтах — провайдеры подключаются через реестр.

## Быстрый старт

1. Создай бота и получи токен — см. [telegram_setup.md](telegram_setup.md).
2. Скопируй `.env.example` в `.env` и вставь токен:
   ```bash
   cp .env.example .env
   # отредактируй TELEGRAM_BOT_TOKEN
   ```
3. При необходимости поправь параметры в `setup.yaml`
   (см. [configuration.md](configuration.md)).
4. Запусти:
   ```bash
   docker compose up -d
   ```
5. Открой бота в Telegram, отправь `/start`.

Логи: `docker compose logs -f`. Остановка: `docker compose down`.

## Команды бота

| Команда | Действие |
|---------|----------|
| `/start` | Приветствие и краткое описание |
| `/help` | Справка по командам |
| `/add` | Добавить отслеживание (пошагово; города пишутся словами, коды аэропортов определяются автоматически) |
| `/list` | Список отслеживаний + кнопки удаления |
| `/cancel` | Отменить текущий ввод |

## Важно про парсинг

Провайдеры `s7` и `redwing` поставляются как **рабочие шаблоны**: структура,
построение URL и каркас парсинга готовы, но конкретные CSS-селекторы / эндпоинты
API помечены `TODO` и требуют донастройки под живую вёрстку сайта. Это сделано
осознанно — реальные сайты меняются и защищены от ботов, а парсинг может нарушать
их условия использования. Как адаптировать провайдер — см.
[providers.md](providers.md).
